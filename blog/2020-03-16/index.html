テーリの世界紹介 (Part 2) - ハイジャックについて

テーリの世界紹介 (Part 1) - 種族について の続きです。

この記事ではテーリの能力のうち、ハイジャックというものについて紹介します。

背景: ゲーム用の設定としてのハイジャック

テーリというキャラクターは、もともとスパイやハッカーというわけではありませんでした。 なんでもそつなくこなす万能キャラというイメージはあり情報技術についても不得手なわけではありませんでしたが、 それが本業だということになったのは、現在制作中の Hijack のゲームシステムが考えられ、 その主人公として彼女が抜擢されたあとのことです。

つまり歴史的にはハイジャックというシステムとテーリというキャラクターは独立して存在し、 Hijack の開発が始まったから彼女がハイジャックするキャラクターになったのだ、 ということになります。

なので、ハイジャックはどちらかといえばゲームシステムにとって都合のいい設定になっています。 そのためそれをもとにした SS などでは、少々アンリアルな描写にもなってしまいますが、 ご愛嬌です。

ホロイオ

ホロイオはテーリが両腕につけているブレスレットで、 Holo I/O の略です。 Holo I/O で動作するプログラムは C 言語で holoio.h を include することで開発することができるようです。 

#include <holoio.h>
int main(void)
{
  return 0;
}

ホロイオには入力デバイスとしての側面と出力デバイスとしての側面があります。 ゆえに Input/Output と呼ぶのです。

Input 部分は腕輪の手首側についた円筒状のデバイスで手の動きを AI で解析することで キーボード・マウス入力が可能です。 Output 部分は同じく円筒状のデバイスより単色のホログラムが出力できます (256 色が選択可能ですが、同時に発色できるのは 1 色のみ)。

このうち、最初期は Input がホロイオの主力機能でした。 Output は株主総会で搭載が決定した「おまけ」機能です。 ホログラムが出力できたほうが売れるだろうという経営判断で搭載されました。

このホロイオがテーリがハイジャックをするときの最大の外見的特徴です。

ハイジャックでできること

ハイジャックは敵のコンピュータ・システムを攻撃し、その権限を掌握することができます。

テーリはハイジャックのことを、ハッキングとは呼びませんしクラッキングとも呼びません。 あくまでハイジャックと呼びます。

基本的に、ハイジャックが成功した場合、そのシステムで可能なことをすべておこなうことができます。

ハイジャックの条件

ハイジャックを成功させるためには条件があり、それは対象のシステムに脆弱性（セキュリティホール）が存在することです。 脆弱性のないシステムをハイジャックすることはできません。 テーリといえども、脆弱性のないシステムをハイジャックすることは不可能です。

ではなぜテーリがほとんどのシステムをハイジャックできるのかというと、 ほとんどのシステムには脆弱性があるからです。

システムに脆弱性が存在する理由はさまざまです。 単純にセキュリティのアップデートを怠っている場合がほとんどです。 ときにはゼロデイ脆弱性というものもありますが、民間のシステムの多くは、 そんなものを使わずとも簡単にハイジャックできてしまいます。

特別なセキュリティシステムを導入していなくとも、 きちんとシステムをアップデートしているシステムならば、 ハイジャックするのはそう簡単ではありません。 ゼロデイ脆弱性の知識が必要です。 こういった知識を入手するのは簡単ではありません。並のハッカーにはまず無理です。

しかしテーリにはそれができます。 なぜかというと、彼女は NSA の諜報員で、 NSA が収集しているゼロデイ脆弱性のリスト（の一部）にアクセスできる権限をもつからです。

このゼロデイ脆弱性についての知識がテーリと並のハッカーの最大の違いです。

（なお、テーリはこのゼロデイ脆弱性のリストの存在は悪だと認識しており、 このリストの存在を知ったことがきっかけで、 このリストの脆弱性をすべて報告してこの世から消し去るためにスパイになりました。 いわば闇の力であり、闇の力に対抗するためには闇の力が必要、 というのが彼女がスパイをしている理由です）

その他の技術に関しては、テーリはあまり特殊なことはしていません。 ごくふつうのプログラマと似たようなことを粛々とこなしています。

このようにこの世界では、対象のシステムに存在する脆弱性の知識が多ければ多いほど、 ハイジャックできる対象が多いということになります。

テーリは NSA の所有するゼロデイ脆弱性のリスト以外にも、 日夜報告される脆弱性の知識をキャッチアップしています。 もし彼女が NSA の諜報員でなくとも、それらの（非ゼロデイ）脆弱性の知識だけで民間のシステムは おおよそすべてハイジャックできます。 それに加えて彼女には NSA の所有しているゼロデイ脆弱性のリストの知識まであります。 それゆえあれほどまでに完璧にハイジャックを成功させることができるのです。

未知のシステムへの対策

ですが、テーリでも知らない脆弱性はあります。 とくにテーリはスパイですから、仮想敵国の新型兵器と対峙する場合など、いっさいの前知識なしに 戦わなければならないことも多いのです。

そういう場合、テーリは最初、その新型兵器をハイジャックできない場合が多いのです。 脆弱性が存在してもどんな脆弱性が存在するかわからないのですから、仕方ありません。

ですがひとまず 1 体その新型兵器を倒して鹵獲し解析することで脆弱性の情報を知ることができます。 これは一般スキャンという過程でおこなわれます。

ハイジャックに対する免疫

ハイジャックへの基本的な防衛方法はセキュリティをアップデートすることですが、 それですべてのハイジャックを防げるわけではありません。脆弱性は常に見つかっています。

そこであえて敵に 1 度ハイジャックを成功させ、正常な状態との差異からそれを検知することで、 システム自体がセキュリティを自力でアップデートしようという試みもあります。 それはまるでハイジャックに対する免疫系のようなふるまいを見せます。

このようなシステムは軍用のロボットに多いのです。 1 体ハイジャックされても、それに対する免疫を獲得してしかもそれを P2P 通信で共有することで、 ハイジャックに対する強力な免疫系を獲得するという、非常に高度なシステムです。

しかしそれも万能ではありません。それを逆手にとって、 ウイルスを免疫に偽装して P2P 通信に紛れ込ませ、敵のロボット全体をハイジャックしてしまうという荒業もあるのです。

バイオジャックについて

人間を含めて生命体にはコンピュータにおける脆弱性のようなものが存在し、 生体ハイジャック（バイオジャック）が可能なことは、何度も実験によって確かめられてきました。 初期のそれは昆虫における研究が代表的です。当初、それはあくまで昆虫におけるもののみが可能なのであって、 人間のような "高次な" 生命体では不可能と考えられてきましたが、 最近雲行きが変わりつつあります。

昆虫におけるバイオジャックはテーリの時代ではさまざまな応用がなされています。 またより複雑な生命体においても、バイオジャックの可能性を示す研究結果は数多く存在します。

テーリも興味津々で論文を読み漁っています。

彼女にとっては、ハイジャックが "実践的な" 技術だとすればバイオジャックは "最先端の" 技術です。 それを主軸に戦うことはまだあまりありませんが、 追いつめられたときには昆虫をバイオジャックして仲間とコンタクトを図るなど、 常に最先端技術にも目を光らせています。

ゲームシステムとしては、バイオジャックというのはできないほうがゲーム性はあがります。 ハイジャックできるドローンに対してハイジャックできない生物というのはわかりやすいバランスです。

なのでバイオジャックというのはもともと "まったくできない" 設定にしようと思っていたのですが、 最近は昆虫くらいならできてもおかしくないんじゃないかなと感じていて、 現在テーリは "多少は" バイオジャックができるのかもしれない、という感じになっています。

バイオジャックについてはハイジャックとはまた別の話ですので今回はこのへんで。
